## Детали

### BUG-1 — Logout button missing in /profile

**Проблема:** страница `/profile` не содержала кнопки выхода из аккаунта.
Пользователь мог выйти только через sidebar или другие страницы.

**Исправление:** в панели «Профиль» добавлена секция «Сессия» с кнопкой
«Выйти из аккаунта» (вызывает `window.Common.handleLogout()`).

---

### BUG-2 — Admin cannot change user passwords

**Проблема:** страница `/admin` позволяла только создавать и удалять пользователей.
Не было способа сменить пароль существующего пользователя через UI администратора.

**Исправление:**
- Новый API-эндпоинт `POST /api/admin/change_password/<user_id>`
- Кнопка «Пароль» для каждого пользователя в таблице (кроме root)
- Bootstrap-модал с полем нового пароля и подтверждением
- Запись события `admin_change_password` в AuditLog

**Ограничения:**
- Пароль пользователя `root` защищён от изменения (403)
- Минимальная длина пароля: 8 символов (по `MIN_PASSWORD_LEN`)
