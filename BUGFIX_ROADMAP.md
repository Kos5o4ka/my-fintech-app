## Детали по новым багам (v1.4.0)

### BUG-3 — Missing coupon income for sold bonds in tax report
**Проблема:** В расчете `coupon_income` за отчетный год учитываются только бумаги, находящиеся в списке `active_bonds` (активные на данный момент). Если пользователь владел облигацией, получал по ней купоны в течение года, а затем продал ее, эта облигация переходит в архив (`sold_bonds`). В результате купонный доход по ней полностью выпадает из расчета НДФЛ, занижая налоговую базу.

**Исправление:** Объединить сканирование активных и проданных облигаций за отчетный год с фильтрацией по дате выплаты купона.

---

### BUG-4 — OTP 2FA brute-force vulnerability
**Проблема:** Метод `verify_otp` удаляет временный OTP-код из кэша (`cache.delete`) только в случае успешного совпадения (`if is_valid:`). Если злоумышленник пытается подобрать 6-значный цифровой код (всего 1 млн комбинаций), он может отправлять сотни запросов в секунду. Так как код не сгорает при ошибке ввода, brute-force атака легко подберет код за 5 минут жизни токена.

**Исправление:** Сделать OTP-код строго одноразовым! Код должен удаляться из кэша незамедлительно при первой же попытке проверки, независимо от того, верный он или нет.

---

### BUG-5 — Unauthenticated Telegram Webhook (Spoofing)
**Проблема:** Эндпоинт `/api/telegram/webhook` является публичным и не имеет авторизации. Любой сторонний клиент может отправлять поддельные HTTP POST запросы, эмулируя сообщения от Telegram (например, привязывая чужие chat_id, перехватывая сессии 2FA или генерируя ложные команды).

**Исправление:** Использовать секретный токен в URL вебхука, известный только Telegram и нашему серверу (`/api/telegram/webhook/<secret_token>`).

---

### BUG-6 — Math error in weighted portfolio YTM
**Проблема:** В текущей формуле сумма весов YTM делится на общую стоимость всего портфеля. Если в портфеле есть бумага без YTM (например, ОФЗ-ИН или бумага с неполными данными на MOEX), ее стоимость все равно прибавляется к знаменателю, что искусственно занижает общий YTM портфеля.

**Исправление:** Делить сумму взвешенных доходностей только на суммарную стоимость тех позиций, у которых есть валидный YTM.

---

### BUG-7 — Screener "Limit before Filter" bug
**Проблема:** Запрос к MOEX ISS API выполняется с жестким лимитом (по умолчанию 100-200 бумаг), после чего к списку локально применяются фильтры. Из-за этого подходящие пользователю бумаги, находящиеся глубже в выдаче MOEX, полностью отсекаются.

**Исправление:** Добавить базовые фильтры (YTM, срок) непосредственно в параметры запроса к MOEX ISS API или существенно увеличить пакет первичной выборки.

## Чеклист тестирования после деплоя (v1.4.0)

- [x] Вход с неверным OTP-кодом 2FA возвращает ошибку 401, и повторный ввод (даже верного кода) блокируется до генерации нового токена.
- [x] Вебхук `/api/telegram/webhook` без секретного ключа в URL возвращает ошибку 403.
- [x] Купонный доход по проданным в отчётном году бумагам корректно отображается в налоговом отчёте.
- [x] Средневзвешенный YTM портфеля рассчитывается корректно при наличии бумаг с нулевым/отсутствующим YTM.
- [x] Скринер выдаёт релевантные облигации, даже если они не входят в топ-100 по умолчанию на MOEX.
